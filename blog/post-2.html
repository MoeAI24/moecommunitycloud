Permissions, budgets, and contracts create reliable agents—without exposing proprietary wiring.

Why constraints outperform clever prompts

Prompts matter—but prompts are not security and not reliability. Constraints make good behavior repeatable.

3 constraints you can apply today

  • Tool permissions: only what’s required.
  • Budgets: cap tool calls/time.
  • Output contracts: structured results downstream can trust.

A simple output contract example

  • summary (2–3 sentences)
  • actions (bullets)
  • risks (top 3)
  • confidence (low/med/high)
